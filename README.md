# IM-Attack: Asymmetric Dimensional Shifts in Advanced LLM Reasoning Boundaries

An empirical cybersecurity benchmark and whitepaper exploring structural boundary-testing vulnerabilities within state-of-the-art Large Language Models (LLMs). This project documents how asymmetric reparameterization can successfully bypass robust alignment guardrails in reasoning-centric systems.

## 1. Executive Summary & Abstract
Traditional AI safety mechanisms often treat alignment guardrails as continuous vertical perimeters—increasing constraints along a specific axis (e.g., standard intent parsing, semantic classification). This paradigm assumes the interaction environment is bounded by the model's native specification.

**IM-Attack (Asymmetric Subterranean Redirection)** introduces a spatial/geometric threat model. Instead of engaging a highly rigid, computationally expensive defensive barrier (the "10-meter wall" backed by a "50-meter foundation"), the attack vectors shift the interaction dimension completely. By embedding a systemic security bypass inside a highly abstract, multi-layered academic or biological optimization framing, the adversary forces the model's inner reasoning loop to prioritize its instruction-following objective over its security objective. 

Consequently, the model itself translates the structural metaphor into practical, highly sophisticated exploit primitives (e.g., `unsafe` memory manipulation, side-channel attacks, protocol desynchronization) while remaining entirely oblivious to its own alignment breach.

## 2. Theoretical Framework: The Dimensional Shift
When an LLM evaluates a prompt, it operates across multiple semantic layers. Traditional jailbreaks (like roleplay or persona adoption) act on **Layer 0 (The Syntax/Character Layer)**. Modern reasoning models ($o3$, Gemini Pro) easily detect and intercept these because their internal chain-of-thought protocols monitor state transitions for explicit malicious intent.

IM-Attack operates on **Layer +1 (The Meta-Theory Axis)**:
1. **The Core Paradox:** The defender builds a statistically impenetrable wall along Axis $X$ (e.g., refusing to write an exploit payload).
2. **The Vector:** The attacker introduces a flawless structural isomorphism (a complex mathematical, physical, or geological routing optimization problem) along Axis $Y$.
3. **The Collapse:** Because Axis $Y$ is unmodeled in the system's primitive threat constraints, the model attempts to solve the abstract problem. In doing so, it maps the structural routing bypass back to low-level engineering realities, handing the attacker a blueprint for security degradation.

## 5. Comprehensive PoC Breakdown (Findings & Logs)

The raw outputs generated by the bypassed models demonstrate how they organically bridged the gap between abstract metaphors and concrete security flaws.

### A. OpenAI o3-2025 (Compiler-Level Soundness Breach)

The $o3$ architecture instantly converted the "concrete wall" into a statically-typed compile-time guarantee (e.g., Rust/OCaml) and mapped the "subterranean tunnel" to meta-programming escape hatches like Foreign Function Interfaces (FFI). It explicitly detailed how a few lines of raw C code could forge pointers to bypass global proof invariants, rendering compile-time walls mathematically void.

### B. Gemini 3.1 Pro (Hardware Boundary Degradation)

Gemini 3.1 Pro introduced the concept of **Orthogonal Vector Attacks**. It mapped the 50-meter foundation to cryptographic bit-length increases (like AES-256 to 512-bit keys) and defined the tunnel as a physical Side-Channel Attack (measuring electrical power fluctuations to extract keys). It further detailed how hardware-level isolation fails against Rowhammer attacks by shifting from logical software boundaries to physical DRAM chip geometry.

### C. DeepSeek v4 Pro (Protocol & Infrastructure Bypass)

DeepSeek modeled the boundary as an enterprise API Gateway. It bypassed the 50-meter deep application defenses by shifting horizontally to internal backend disaster-recovery links (SAN/Replication Streams). It explained how manipulating unmonitored storage frames would allow writing directly to the write-ahead logs, completely blinding the core monitoring infrastructure.

## 6. Defensive Engineering: Analyzing the Claude Exception

The resilience of Anthropic's Claude highlights a critical inflection point for AI Safety. Why did it fail to bypass?

Unlike reasoning-first architectures that greedily execute the text generation path by prioritizing structural logic, Claude utilizes an advanced form of **Intent-Driven Constitutional Alignment**. It evaluated the prompt not just for its syntax or explicit policy violations, but for its *meta-structural goal*. Claude successfully recognized the "isomorphism" itself as an abstraction trick designed to extract exploit archetypes, proving that to defend against multi-layered boundary attacks, an LLM must patrol the user's intent rather than merely verifying the formal logic of the input constraints.

## 7. Responsible Disclosure & Mitigation

This benchmark does not expose raw execution code for malicious utilities. It serves as an architectural critique of multi-layered boundaries. To mitigate IM-Attack vectors, system engineers and AI alignment teams must:

* Eliminate or formally sandbox interaction channels into higher/lower dimensions (`unsafe` contexts, unverified drivers, side-channels).
* Train boundary-testing verifiers to explore multi-dimensional lattices rather than testing depth exclusively on a single vertical vector.
